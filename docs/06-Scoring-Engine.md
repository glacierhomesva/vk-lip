# Scoring Engine

**Project:** VK Land Intelligence Platform (VK-LIP)

**Version:** 0.1

**Status:** Draft

**Owner:** Viraj Kothari

**Technical Lead:** ChatGPT

**Last Updated:** 2026-06-30

---

# Purpose

The Scoring Engine transforms public real estate data into acquisition intelligence.

Rather than presenting raw parcel information, the engine evaluates every property using standardized scoring models and recommends the next best action.

The Scoring Engine is the core intellectual property of VK-LIP.

---

# Objectives

The engine should answer four questions for every parcel.

1. Is this a good investment?
2. Can Glacier Homes successfully develop it?
3. Is the owner likely to sell?
4. Should action be taken now?

---

# Core Scores

Every parcel receives four independent scores.

These scores should never be combined directly inside the database.

The overall recommendation is calculated by the Business Rules Engine.

---

## 1. VK Score™

Purpose

Overall acquisition attractiveness.

Range

0–100

Higher is better.

Inputs

- Zoning
- Lot Size
- Parcel Geometry
- Assessment History
- Sales History
- Neighborhood
- Strategic Investment Area
- Redevelopment Potential
- Developer Pressure
- Infrastructure
- Risk Adjustments

Output

Single numeric score.

---

## 2. Builder Score™

Purpose

Measures construction feasibility.

Question answered

Can Glacier Homes realistically execute this project?

Inputs

- Existing Structure
- Demolition Complexity
- Utilities
- Access
- Topography
- Lot Width
- Lot Depth
- Corner Lot
- Easements
- Tree Constraints
- Flood Risk (future)

Output

0–100

---

## 3. Seller Probability™

Purpose

Predict likelihood of acquisition.

Question answered

How likely is the owner to sell?

Inputs

- Years Owned
- Owner Occupied
- Absentee Owner
- LLC
- Trust
- Estate
- Owner Age (future)
- Mailing Distance
- Recent Refinancing (future)
- Tax Delinquency (future)
- Neighbor Redevelopment

Output

0–100

---

## 4. Momentum Score™

Purpose

Measures market activity surrounding the parcel.

Question answered

Is redevelopment already happening nearby?

Inputs

- Developer Ownership
- Recent Sales
- Building Permits
- Demolition Permits
- Rezoning
- New Construction
- Average Appreciation

Output

0–100

---

# Business Rules Engine

Scores alone do not determine recommendations.

Business Rules evaluate combinations.

Example

IF

VK Score > 90

AND

Seller Probability > 80

THEN

Recommendation

Call Owner

---

Example

IF

Builder Score < 40

Recommendation

Do Not Pursue

---

Example

IF

Momentum Score > 90

AND

Seller Probability < 20

Recommendation

Monitor Quarterly

---

# Recommendation Categories

Every parcel receives one recommendation.

- BUY NOW
- CALL OWNER
- MAIL CAMPAIGN
- DRIVE PROPERTY
- MONITOR
- IGNORE
- UNDERWRITE
- ASSEMBLE

---

# Score Versioning

Every score calculation stores

- Model Version
- Calculation Date
- Calculation Duration
- Input Dataset Version

This allows comparison between scoring models over time.

---

# Score Recalculation

Scores should be recalculated when

- New assessment imported
- Sales imported
- Ownership changes
- Permit activity changes
- Zoning changes
- Manual recalculation requested

---

# Confidence Score

Every score includes confidence.

Range

0–100

Purpose

Communicate data quality.

Example

Seller Probability

92

Confidence

45

Meaning

High prediction

Limited supporting data

---

# Explainability

Every score must be explainable.

Users should never see

VK Score = 91

without understanding why.

The API should return

- Top Positive Factors
- Top Negative Factors

Example

Positive

+ Large Lot
+ SIA
+ Corner Lot

Negative

- Recent Sale
- Small Frontage

---

# Performance Requirements

Full city recalculation

< 10 minutes

Single parcel recalculation

< 1 second

---

# Success Criteria

The scoring engine succeeds when

Two experienced developers reviewing the same parcel would generally agree with the recommendation generated by VK-LIP.

Perfect accuracy is not required.

Consistency and explainability are required.
